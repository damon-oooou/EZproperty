## 1. 登录拿 token（后续命令都用）

```powershell
$auth = Invoke-RestMethod -Method Post -Uri "http://localhost:8080/api/auth/login" `
  -ContentType "application/json" `
  -Body (@{ email = "<你的邮箱>"; password = "<密码>" } | ConvertTo-Json)
$H = @{ Authorization = "Bearer $($auth.token)" }
```

建 property / inspection / 取 roomId（也可直接在 UI 操作后从地址栏读 id）：

```powershell
$prop = Invoke-RestMethod -Method Post -Uri "http://localhost:8080/api/properties" -Headers $H `
  -ContentType "application/json" -Body (@{ address = "1 Test St"; type = "HOUSE" } | ConvertTo-Json)
$insp = Invoke-RestMethod -Method Post -Uri "http://localhost:8080/api/properties/$($prop.id)/inspections" -Headers $H `
  -ContentType "application/json" -Body (@{ type = "ENTRY"; inspectionDate = "2026-07-13"; inheritFromPrevious = $false } | ConvertTo-Json)
$rooms = Invoke-RestMethod -Uri "http://localhost:8080/api/inspections/$($insp.id)/rooms" -Headers $H
$roomId = $rooms[0].id
```

上传辅助函数（PowerShell 5/7 通用 multipart）：

```powershell
function Upload-Photo($inspId, $roomId, $file) {
  curl.exe -s -X POST "http://localhost:8080/api/inspections/$inspId/rooms/$roomId/photos" `
    -H "Authorization: Bearer $($auth.token)" -F "files=@$file"
}
```

## 发现的问题记录（backlog，不顺手修）

- 规格书 3.2 写 `max-request-size=60MB`，但已交付的 v0.5.2 代码与 README 均为 100MB。
  两处不一致，留待所有者拍板（如需改为 60MB 是一行 yml）。
- 规格书 3.1 要求批量上传"单文件不合规不导致整批失败"，已交付的 v0.5.2 实现为
  先整批校验、任一不合规整批 400（含明确文件名）。阶段 A 沿用该已交付行为，
  管线解码失败同样整批拒收 + 报文件名（保证"任一档失败整体回滚、无孤儿文件"更严格成立）。
